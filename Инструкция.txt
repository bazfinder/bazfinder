Это общие команды Git, используемые в различных ситуациях:

запустите рабочую область (см. Также: git help tutorial)
   clone             Клонирование репозитория в новый каталог
   init              Создайте пустой репозиторий Git или повторно инициализируйте существующий

работа над текущим изменением (см. Также: git help everyday)
   add               Добавление содержимого файла в индекс
   mv                Перемещение или переименование файла, каталога или символической ссылки
   restore           Восстановление файлов рабочего дерева
   rm                Удаление файлов из рабочего дерева и из индекса
   sparse-checkout   Инициализации и изменения незначительные-оформить заказ

изучите историю и состояние (см. Также: git help revisions)
   bisect            Используйте двоичный поиск, чтобы найти коммит, который ввел ошибку
   diff              Показывать изменения между коммитами, коммитом и рабочим деревом и т. д
   grep              Печать линий, соответствующих шаблону
   log               Показать журналы фиксации
   show              Показать различные типы объектов
   status            Показать состояние рабочего дерева

растите, отмечайте и настраивайте свою общую историю
   branch            Список, создание или удаление ветвей
   commit            Запись изменений в репозиторий
   merge             Объединение двух или более историй развития вместе
   rebase            Повторно примените фиксации поверх другого базового наконечника
   reset             Сброс текущей головы в заданное состояние
   switch            Переключатель филиалы
   tag               Создавать, просматривать, удалять или проверить тега объекта подписан с GPG

совместная работа (see also: git help workflows)
   fetch             Загрузка объектов и ссылок из другого репозитория
   pull              Извлечение из другого репозитория или локальной ветви и интеграция с ними
   push              Обновите удаленные ссылки вместе со связанными объектами

'git help -a' и 'git help -g'перечислите доступные подкоманды и некоторые из них
понятие руководства. Смотри 'git help <command>' или 'git help <concept>'
читать о конкретной команды или концепции.
Смотри 'git help git' для получения общего обзора системы.
